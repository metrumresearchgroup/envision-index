Customize Envision Index page with envision-manifest

The envision dashboard page is how you access your shiny-server apps. It provides a list of apps found at /data/shiny-server, with a link to each. This is also where you can view the logs for your applications. Until now, envision users were not able to customize this page. This is now possible, via the envision-manifest.

The envision-manifest allows developers to customize the index page by enabling icons, app descriptions, etc. This purpose of this document is to help developers get up and running with an envision-manifest.

The envision-manifest is a folder that contains meta data about the app and (optionally) an image file to use as the icon. The quickest way to get up and running is to look at the example that is deployed with an Envision session, stored with the "hello" app.

Location

The envision-manifest is stored at the same level as the app (server.R and ui.R or app.R). To be useful, the envision-manifest must contain a file called "app-info.yaml". Additionally, an icon may be supplied in this folder (and will be specified in app-info.yaml).

app-info.yaml

app-info.yaml allows for the following options:

  • icon: File name of icon (must be located in envision-manifest folder). If not provided, the default is the Metrum icon.
  • name: Display name for app. If not provided, the default is the app's folder name.
  • description: The app's description.
  • author: The app's author. If not provided, the default is generated using the unix command "uname" on the app's folder.
  • log_button: Should access to the logs be displayed for this application? If not provided, the default is TRUE.
